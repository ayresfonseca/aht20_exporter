# aht20_exporter — Agent Instructions

Prometheus exporter for AHT20 temperature/humidity sensors connected via a TCA9548A I2C multiplexer on Raspberry Pi.

## Key Facts

- Single-file project: all logic lives in [`aht20_exporter.py`](aht20_exporter.py)
- Python ≥ 3.11, managed with **Poetry ≥ 2.0**
- Target platform: **Raspberry Pi 3 Model B Rev 1.2**, Raspbian 12.14, Python 3.11.2
- Metrics exposed at `http://localhost:9101/metrics`
- Hardware-only: `board`, `adafruit_tca9548a`, and `adafruit_ahtx0` require physical I2C hardware — unit tests cannot be run without a Raspberry Pi with AHT20 sensors attached

## Common Commands

```sh
poetry install                    # install all dependencies
poetry run aht20_exporter         # run with default 3 sensors
poetry run aht20_exporter --sensors 4   # override sensor count

# Linting / formatting (CI mirrors these)
poetry run black .
flake8 --extend-ignore=E501 $(git ls-files '*.py')
pylint --disable=C0301 $(git ls-files '*.py')
```

## Conventions

- Sensor count is configurable via `--sensors` CLI flag or `AHT20_SENSOR_COUNT` env var (default: 3)
- Prometheus metric names: `aht20_temperature_celsius` and `aht20_humidity`, labeled `sensor0`, `sensor1`, …
- Sensor values rounded to 1 decimal place before being set as Gauge values
- `requirements.txt` is auto-generated by `pre-commit` via `poetry-plugin-export` — **do not edit it manually**
- pre-commit hooks enforce: trailing whitespace, EOF newline, large-file guard, `poetry check`, `poetry lock`, and `requirements.txt` export

## Dependency Management

- Add/update deps in `pyproject.toml` via `poetry add` / `poetry remove`
- After changing deps, run `poetry lock` and commit both `pyproject.toml` and `poetry.lock`; `requirements.txt` will be regenerated automatically by pre-commit
